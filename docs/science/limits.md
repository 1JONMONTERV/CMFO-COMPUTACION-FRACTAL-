# Límites y Falsabilidad Científica de CMFO

> **Propósito:** Este documento establece los límites operativos de la teoría CMFO y define las condiciones bajo las cuales la teoría podría demostrarse falsa, cumpliendo con el criterio de demarcación científica de Popper.

## 1. Alcance Operativo

### Dónde funciona CMFO
CMFO está diseñado específicamente para:
*   **Sistemas de Alta Entropía:** Modelado de caos, turbulencia y mercados.
*   **Compresión Semántica:** Reducción de grandes volúmenes de datos mediante fractales.
*   **Lógica Difusa/Continua:** Toma de decisiones en entornos de incertidumbre.

### Dónde NO es óptimo
CMFO no pretende reemplazar a la computación clásica en:
*   **Aritmética Discreta Simple:** Para sumar $2+2$, una ALU de silicio clásica es insuperable. CMFO introduciría overhead innecesario.
*   **Criptografía de Curva Elíptica Estándar:** CMFO utiliza su propia criptografía geométrica; no está diseñado para emular RSA/ECC eficientemente.

## 2. Criterios de Falsabilidad

La teoría CMFO se considerará **refutada** si se demuestra cualquiera de las siguientes proposiciones:

1.  **Divergencia de Atractores:** Si se encuentra un tensor de entrada válido tal que la función $\Gamma_\varphi$ no converja a un atractor estable ni entre en un ciclo límite predecible en $t \to \infty$.
2.  **Ruptura de Isomorfismo:** Si se demuestra formalmente que existe una operación booleana fundamental que **no** puede ser representada como un mapeo continuo en $\mathbb{R}^7$.
3.  **Ineficiencia Termodinámica:** Si el coste energético de computar un ciclo fractal $\Gamma$ supera teóricamente el límite de Landauer para la información equivalente procesada.
